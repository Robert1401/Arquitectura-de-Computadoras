1.1 Modelos de arquitectura de computadoras

1.1.1 Clásicas:

  • Arquitectura de Von Neumann: Basada en el modelo propuesto por John von Neumann, donde las instrucciones y los datos se almacenan en la misma memoria y se manipulan mediante la misma unidad de procesamiento.
  • Arquitectura de Harvard: Se caracteriza por tener memorias separadas para datos e instrucciones, lo que permite un acceso simultáneo a ambos tipos de datos y mejora el rendimiento en ciertos escenarios.

1.1.2 Segmentadas:

  • Pipelining: Divide el procesamiento de instrucciones en etapas para que múltiples instrucciones se ejecuten simultáneamente, aumentando la velocidad de ejecución.
  • Superescalar: Permite la ejecución de múltiples instrucciones en paralelo utilizando múltiples unidades de ejecución.

1.1.3 De Multiprocesamiento:

  • Multiprocesamiento Simétrico (SMP): Todos los procesadores tienen acceso uniforme a la memoria compartida y ejecutan el mismo sistema operativo, lo que permite una distribución equitativa de la carga de trabajo.
  • Multiprocesamiento Asimétrico (ASMP): Los procesadores tienen diferentes roles y pueden ejecutar diferentes sistemas operativos, lo que permite una mayor flexibilidad pero requiere una gestión más compleja.

1.2 Análisis de los Componentes

1.2.1 Arquitecturas:

CISC (Complex Instruction Set Computer):

  • Características:
    • Conjunto de instrucciones complejo y variado.
    • Las instrucciones pueden realizar múltiples operaciones.
    • Énfasis en el software para optimización.
    • Ejemplos: x86 de Intel, x86-64 (AMD64).
  • Ventajas:
    • Menor código de programa.
    • Mejor para lenguajes de alto nivel.
    • Eficiencia en programas complejos.
  • Desventajas:
    • Menor rendimiento por ciclo.
    • Mayor complejidad de diseño.
    • Más difícil de optimizar.

RISC (Reduced Instruction Set Computer):

  • Características:
    • Conjunto de instrucciones reducido y simples.
    • Énfasis en la ejecución eficiente de un conjunto limitado de operaciones.
    • Diseño de hardware más simple y eficiente.
    • Ejemplos: ARM, MIPS, PowerPC.
  • Ventajas:
    • Mayor rendimiento por ciclo.
    • Diseño de hardware más simple.
    • Más fácil de optimizar.
  • Desventajas:
    • Más líneas de código.
    • Menos flexibilidad.
    • Menor compatibilidad con lenguajes de alto nivel.

1.2.1.1 Unidad Central de Procesamiento (CPU):

  • Control Unit (Unidad de Control): Coordina las operaciones del procesador y controla la ejecución de instrucciones.
  • Arithmetic Logic Unit (Unidad Aritmético Lógica): Realiza operaciones aritméticas y lógicas en los datos.

1.2.1.2 Unidad Aritmética Lógica (ALU):

  • Suma, resta, multiplicación, división, operaciones lógicas como AND, OR, NOT.

1.2.1.3 Registros:

  • Program Counter (Contador de Programa): Almacena la dirección de la próxima instrucción a ejecutar.
  • Instruction Register (Registro de Instrucción): Almacena la instrucción actualmente en ejecución.
  • General Purpose Registers (Registros de Propósito General): Almacenan datos temporales durante la ejecución de programas.

1.2.1.4 Buses:

  • Bus de Datos: Transfiere datos entre la CPU, la memoria y los dispositivos de E/S.
  • Bus de Direcciones: Transporta direcciones de memoria para acceder a datos e instrucciones.
  • Bus de Control: Coordina las señales de control entre los componentes del sistema.

1.2.2 Memoria

1.2.2.1 Conceptos Básicos del Manejo de la Memoria:

  • Localidad Temporal: La tendencia de acceder a los mismos datos repetidamente en un corto período de tiempo.
  • Localidad Espacial: La tendencia de acceder a datos cercanos en direcciones de memoria cercanas.

1.2.2.2 Memoria Principal:

  • RAM (Random Access Memory): Almacena datos y programas en ejecución de forma temporal.
  • ROM (Read-Only Memory): Almacena instrucciones de arranque y firmware.

1.2.2.3 Memoria Caché:

  • Niveles de Caché: L1, L2, L3; cada nivel proporciona un grado diferente de cercanía a la CPU y velocidad de acceso.

‍

1.2.3 Manejo de la Entrada/Salida

1.2.3.1 Módulos de Entrada/Salida:

  • Controladores: Traducen las operaciones de E/S a comandos que pueden entender los dispositivos.
  • Puertos: Interfaces físicas que permiten la conexión de dispositivos periféricos.

1.2.3.2 Entrada/Salida Programada:

  • La CPU controla directamente la transferencia de datos entre la memoria y los dispositivos de E/S.

1.2.3.3 Entrada/Salida Mediante Interrupciones:

  • Los dispositivos externos pueden interrumpir la CPU para solicitar atención, lo que permite una gestión eficiente de eventos asíncronos.

1.2.3.4 Acceso Directo a Memoria:

  • Permite que ciertos dispositivos de E/S transfieran datos directamente a la memoria sin intervención de la CPU.

‍

‍

1.2.4 Buses

1.2.4.1 Tipos de Buses:

  • Paralelos: Transmiten varios bits simultáneamente.
  • Seriales: Transmiten un bit a la vez, pero pueden alcanzar velocidades más altas.

1.2.4.2 Estructura de los Buses:

  • Bus Interno: Conecta componentes dentro del sistema.
  • Bus Externo: Conecta la CPU con dispositivos externos.

1.2.4.3 Jerarquías de Buses:

  • Jerarquía de Velocidad: Buses más rápidos cerca de la CPU y buses más lentos para dispositivos periféricos.

‍

1.2.5 Interrupciones

  • Vectores de Interrupción: Tablas que contienen direcciones de rutinas de servicio de interrupción.
  • Prioridades de Interrupción: Determinan el orden en que se manejan múltiples interrupciones.
  • Máscara de Interrupción: Permite desactivar ciertas interrupciones para evitar su procesamiento en momentos críticos.
